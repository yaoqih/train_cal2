# 福州调车业务文档


## 业务总览

### **文档定位**

本文档用于定义福州东车辆段的业务事实、场区主数据、作业能力和标准业务流程，目标是让不熟悉现场的人也能据此建立稳定的领域模型和主数据表。

### **业务背景与目标**

#### **车辆段作业对象**

车辆段是铁路货车的检修场所，车辆会在段内完成预修、调梁、洗罐、抛丸、油漆、大库检修、卸轮装卸等不同业务。

调车作业的核心任务，是使用调车机把车辆从当前股道送到目标股道、目标作业区或目标台位，并在工序完成后继续送往下一目标，直到全部工序结束并出段。

#### **当前业务痛点**

1. 人工编制钩计划耗时长，依赖经验，计划质量波动大。

2. 现场存在错办、挤岔、撞档、多拉少挂等安全风险。

3. 连结员需要频繁行走、摘挂、扳岔和安撤防护装置，作业强度高。

4. 人、车、线、设备、计划之间信息割裂，协同效率低。

#### **建设目标**

1. 安全目标：减少人工错办、挤岔、撞档、多拉少挂和防护装置遗漏风险。

2. 效率目标：缩短钩计划编制时间，降低人工强度，提升人车线协同效率。

#### **相关外部系统**

需求说明提到，后续业务会依赖以下系统提供输入或执行支撑：

1. 全息数字孪生作业管理平台：展示线路、车辆、设备和计划状态。

2. 车辆运行态势感知系统：提供车号、位置、车型、空重、修程等状态数据。

3. 调车安全防护系统：用于路径安全、防护装置和现场风险校验。

4. 电动转辙机远程集中控制系统：用于路径执行和道岔控制。

5. 调车机智能辅助驾驶系统：用于路径导航、计划下发和执行辅助。

### **术语与对象模型**

#### **术语**

|术语|定义|
|---|---|
|股道|可走行、停放或作业的具体线路|
|区域|若干股道组成的业务分区，如预修区、功能作业区、运营能力区|
|作业区|某条股道上真正执行设备作业或台位作业的那一段范围|
|台位|大库、洗罐、油漆、调棚等作业区内的具体停放位|
|临停|仅允许调车过程中短时甩车，不允许长期停留|
|缓存|为后续取送、清线、折返服务的中间停放用途，不是独立场区|
|缓冲区|辆渡区及其横移过渡区域，只承担横移和过渡，不作为普通存车区|
|对位|把车送入特定作业区或特定台位|
|非对位业务|只要求到达指定线或指定能力区，不要求精确落到固定台位|
|可停车有效长度|股道用于停车、留车、容量校验的有效长度，不含警冲标和岔区不可停车部分|
|实际物理距离|车辆运行或倒车时用于判断“走不走得过去、会不会干涉”的分支物理距离，按同一分支下各片段长度求和|
|关门车|不具备接风管/自动制动条件的车辆属性，不是独立车型|

#### **基本关系**

1. 一条股道可属于一个区域。

2. 一条股道可包含零个或多个作业区。

3. 一个作业区可包含零个或多个台位。

4. 当前接口中的车辆属性字段一次只表达一个离散属性值，允许为空，不做多属性并存建模。

#### **主流程**

`扣车计划 -> 存5接车 -> 信息判定 -> 形成本轮调车目标 -> 执行本轮计划 -> 到达目标工序或阶段性停放 -> 工序完成判定 -> 形成下一轮调车目标 -> 多轮循环 -> 全部工序完成 -> 修竣集结 -> 存4北交站`

#### **进段与信息判定**

车辆进入存5后，至少要得到以下业务结论：

1. 车号

2. 修程类别

3. 是否重车

4. 是否称重车

5. 是否关门车

6. 本轮明确调车目标

7. 当前工序完成后是否还存在下一轮工序目标

补充说明：

1. 车辆通常依据扣车计划由车站送入段内 `存5北`、`存5南`。

2. 车辆进段后会先经历预检或信息判定，再形成当轮调车目标。

3. 第一目标由上游业务明确给定，不由本文自动推导。

#### **典型工序流转背景**

下表用于说明福州东车辆段常见业务流，是业务背景，不是算法自动推导规则：

|车辆类别|常见首去向|常见后续流向|常见终点|
|---|---|---|---|
|非罐车功能检修|预修能力组|功能作业区或大库|存4北|
|需要调棚作业车辆|调棚作业区|预修或功能作业区|存4北|
|非临修罐车|洗南|功能作业区或大库|存4北|
|临修罐车|洗南|一般存车区或存4北|存4北|
|营运重车/装卸车|轮|一般存车区或存4北|存4北|

补充说明：

1. `预修`、`调棚`、`抛`、`油` 等工序完成后，车辆通常继续转往大库。

2. 非临修罐车洗罐完成后，通常继续转往大库；临修罐车洗罐完成后可直接转往存4北相关出段流程。

3. 轮线既可接收存5来的营运重车，也可承接大库修竣后的装轮、托运等后续运营作业。

#### **修竣集结与出段**

1. 全部工序完成后，可出段车辆最终统一向存4北集结。

2. 存4南只作为过渡临停，不作为正式出段终点。

3. 关门车在修竣集结及其他相关钩计划中仍需满足专门顺位限制

#### **调车作业窗口**

每天调车时间：

1. 中午 `11:30-14:00`

2. 下午 `18:00-24:00`

## 物理场区与空间数据

### **场区结构与股道主数据**

#### **场区业务分区**

福州东车辆段按业务可分为八类区域：

1. 进段接车区：承接车站送入段内的车辆，作为待分流、待判定和后续形成本轮目标前的接车区域，主要对应 `存5北`、`存5南`。

2. 一般存车区：承担普通停放、缓存和进出作业区前后的过渡停放，主要包括 `存1`、`存2`、`存3`、`机北`、`调北`、`洗北` 以及 `修1库外` 至 `修4库外`。

3. 出段集结区：承担修竣车辆正式集结和交站，正式出段集结线为 `存4北`；`存4南` 不属于正式出段集结区，只作过程临停。

4. 预修区：承担预修能力，主能力线为 `预修`、`机棚`；调棚南端无编号尽头预修段也归入本区，但调棚整条线路不整体归入预修区。

5. 功能作业区：承担调梁、洗罐、抛丸、油漆和大库检修等功能作业，主要包括调棚 `1` 至 `4` 号位、洗南 `1` 至 `3` 号位、`抛`、`油`、`修1库内` 至 `修4库内`；现场不再把段修区、厂修区写成两个固定物理区域。

6. 临停区：只用于调车过程中的临时甩车和阶段性过渡停放，不作为正式终点，主要包括 `临1` 至 `临4` 和 `存4南`。

7. 运营能力区：承担卸轮、装卸、托运等运营作业，当前由 `轮` 单独构成，不并入功能作业区。

8. 联线与辆渡缓冲区：承担线路联通、路径通过和横移过渡，主要包括 `联6`、`联7`、`渡1` 至 `渡13` 及相关缓冲区域，不作为普通存车区。

补充口径：

1. 调棚属于复合线路：`1` 至 `4` 号位是功能作业区内的调梁对位工位，最南尽头是2个预修作业位。

2. 机库不单独列入上述八类区域，按专用线管理；线内包含 `1` 个称重功能位，机库主线既可在调车过程中配合称重，也可作为最终存车线和最终目的地。

#### **股道能力分类**

|分类|典型股道|业务语义|
|---|---|---|
|存车线|存1、存2、存3、存4北、存5北、存5南、机北、调北、调棚、机北3、洗北、修1库外至修4库外|可作为起点、终点、缓存线|
|临停线|机北1、机北2、机南、洗油北、存4南|只能临时甩车，不能作为正式终点|
|作业线|预修、机棚、调棚、洗南、抛、油、轮、修1库内至修4库内|承担预修、功能作业或运营作业|
|走行线|联6、联7、渡1至渡13|只作为路径，禁止停放|
|专用线|机库|调车机停放、称重，也可作为存车线和最终目的地|

## 核心算法与调度约束

### ** 基本作业口径**

#### **调车机位置与作业方向**

1. 调车机固定在车列北侧进行取送作业。

#### **一钩与勾计划**

1. 一“钩”是调机完成一次完整“摘钩/挂钩”的最小作业单元。

2. 勾计划是若干连续“钩”组成的有序调车指令序列，包含作业股道，动作（摘钩，挂钩），数量。

#### **优化目标优先级**

1. 总勾数最少。

2. 道岔扳动次数最少。

3. 路径总代价最小。

#### 求解时间限制

1. 求解时间限制在5分钟

### **车辆分类与业务属性**

#### **车型分类**

开发阶段至少区分：

1. 罐车

2. 棚车

3. 敞车

4. 平车

#### **修程分类**

|修程类别|业务含义|典型周期口径|
|---|---|---|
|段修|常规周期性检修|约 1 至 1\.5 年|
|厂修|周期更长、修程更深|约 2 至 3 年|
|临修|故障处置|不按固定周期，按故障触发，一般不进大库|

### **关键业务属性**

1. 称重车：本轮流程中需要经过机库称重工位。

2. 重车：运营车辆、用于在卸轮线运送工件，影响牵引力以及调车机挂车数量。

3. 关门车的详细顺位限制统一见《福州东调车业务说明》。

### **关门车规则**

|场景|判定条件|约束|
|---|---|---|
|不去 `存4北`|本钩最终目标不是 `存4北`，且看本钩连挂完成后的机后总辆数|机后总辆数不超过 `10` 辆时不额外限制；超过 `10` 辆时，关门车不得位于第一顺位|
|不去 `存4北`|所挂的车辆中包含重车|第一辆车不能是关门车|
|去 `存4北`|本钩最终目标是 `存4北`|关门车在最终摆位结果中不得位于第 `1`、`2`、`3` 顺位|

补充说明：

1. “机后总辆数”按本钩实际连挂完成后的机后车辆总数计算。

2. 若同一钩内同时包含多辆关门车，则每一辆关门车都必须分别满足对应顺位限制。

3. `存4北` 规则仅适用于把 `存4北` 作为调车结束后最终目的地的场景。





### **牵引与长度限制**

V1 统一采用以下硬约束：

1. 纯空车最多 20 辆。

2. 1 辆重车按 4 辆空车折算。

### **临停线路可用长度约束**

1. 可用距离用于计算临停时放入到车辆数量的判断，按输入车辆的车辆长度累计计算。终点是否能够放得下由作业单编制负责。

2. 临时停车可以使用不经过跨越道岔的多条合并线路，输出计划不分勾从而减少勾数。终点停车以及初次取车的勾计划颗粒度必须按照具体的存车线。
### **运行干涉与倒车距离**

1. 若一钩包含倒车、折返或中间换线，必须校验倒车方向上剩余物理距离是否足够容纳“机车 \+ 本钩车辆集合”。

2. 运行和倒车允许经过警冲标和岔区；停车和留车不允许占用该部分。

3. 在约束的时候需要加上调车机自身长度15米。

### **称重约束**

1. 只有带有称重属性的车辆才需要安排称重步骤。具体的过程是需要称重车辆在整个调车作业的过程中某一勾的摘钩目的地时大库，同时称重只有最南边的一个台位，因此在称重时需要称重车在编组的最后一个。

2. 单钩可以挂多辆称重车经过机库，但只有编组最后一辆需要称重的车在本次称重步骤中标记为称重完成；前面的称重车仍需后续再安排称重。

3. 称重只发生在机库称重工位，不改变该车最终业务目标。

### **大库台位分配规则**

#### 车辆长度

|长度条件\(米\)|允许台位|
|---|---|
|`>= 17.6`<br>|仅能用 `301` 至 `305`、`401` 至 `405`；<br>|
|`< 17.6`|优先 `101` 至 `105`、`201` 至 `205`，不足时再到 `301` 至 `305`、`401` 至 `405`；|

#### 修程

|修程|允许台位|
|---|---|
|厂修|只能放到4，5台位|
|段修|能放到所有台位，但是段修车的台位数不能比同线路厂修车台位数大|

#### 已有位置

大库内车如果原地不动，它后面的车也不能动；它后面原本空着的台位，也不能被别的车占用。


### **业务要求**

#### 作业顺序时间考虑

- 在所有目的地位大库的车辆在机走线集结完毕之前，将除大库区域外的调车完成到8\-9成以上。（因为作业前60分钟联7线不开放）（最大预算40勾）

- 单机车去大库里面清理线路和集合（如果一次调不完，可以多次）

- 将大库车辆调到到存4

- 将机走已经编组好的车辆分散到大库的对应位置。

- 最好整个调车只去以及返回大库1次

- 在大库完成调车之后脱轨器就在联7线，而不是全部调车后结束
