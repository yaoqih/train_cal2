# P10 人工差距结构诊断与下一步结构工作

日期：2026-06-29

本文只保留当前仍成立的结论。历史 round21/round22/round48 过程性实验、已经回退的中间抽象、以及过硬但无收益的补丁，不再作为正文保留。

## 1. 当前结论

- 方向是对的：远端业务差距的核心，不是物理路径能不能走，而是远端会话是否真正成了主控结构。
- 短链必须单独成合同，不能当作标准长链的缩短版。
- 物理层已稳定：`hard physical violation = 0`，`unknown route = 0`。
- 但当前还没达到人工计划水平：远端切换仍偏高，短链仍会被标准结构继续展开。

## 4. 已证伪并撤回的实验

- 通用 mixed bundle 中间抽象已撤回。它改变了候选序列，且会把原本可解的案例拖成阻塞。
- 仅把远端预算做成排序惩罚不够。超预算候选仍会被选中，远端切换不会真正收缩。
- 过硬的早进远端、前段强拦截，会直接打碎可解性，不适合作为主线。
- 仅靠局部 `target_move`、`depot_digest` 偏好，无法自然形成人工计划式段落。

这些都不再作为当前主线继续堆叠。

## 5. 结构级根因

1. `RemoteSessionState` 还不是完整业务 session。
   它更多是在统计当前远端状态，还没有先验声明“这一段远端必须如何开始、如何收口”。

2. `short_chain_planlet` 还不是短链合同。
   它只是在候选层提前允许远端动作，没有阻止后续标准链尾巴继续展开。

3. `DEPOT_DIGEST_PLANLET`、`REMOTE_EXCHANGE_PLANLET`、`TARGET_MOVE` 还没有共享同一个 session debt view。
   这会导致同一段远端债务被多个局部结构各自接管，最终形成碎片化切换。

4. `H1-H5` 仍偏软许可，不是债务完成边界。
   这也是为什么同一个案例会先后出现前段、远端、回收、再前段的往返。

## 6. 下一步应该做什么

### 6.1 `RemoteSessionPlanner`

先生成 session contract，再让候选履约。

session 至少要显式包含：

- `session_id`
- `owner`
- `entry`
- `body`
- `exit`
- `allowed_profiles`
- `must_finish_debt`
- `budget`

目标不是再加一个 planlet，而是把远端债务先切成稳定段落。

### 6.2 `SessionOwnedCloseout`

把最后 1-3 个远端目标纳入当前 session 关闭动作里，避免尾部再展开一轮标准链。

### 6.3 `ShortChainCaseContract`

对 `0306W`、`0327W`、`0213W` 这类短链，直接用 case-level 合同：

- 最多一次进入
- 最多一次退出
- 不再追加标准长链尾巴

### 6.4 `SessionOwnedDigest`

让 `DEPOT_DIGEST_PLANLET` 和 `REMOTE_EXCHANGE_PLANLET` 共享同一个远端 session debt view。

它们可以是 session 内部动作，但不能再自己决定整个远端段落的方向。

### 6.5 保留的结构

- 物理校验器
- 路由占用检查
- owner-bound restore
- 现有的 route clear / corridor protection

这些结构是有效的，不该删。

## 7. 当前阶段验收口径

短期目标：

- `completed` 不低于当前基线
- 远端切换 P50 先降到 `<= 5`
- 远端切换 P90 先降到 `<= 7`
- 短链案例不再被标准链尾巴展开

中期目标：

- `completed >= 103 / 113`
- `solver <= manual + 2` 明显高于当前 `67 / 88`
- 远端切换分布逼近人工计划的 4 / 6 双峰
- 总勾数不回退，最好继续下降

## 8. 当前判断

```text
远端会话主控方向仍然成立。
当前缺的不是更多 bundle，而是更硬的 session contract 和短链合同。
下一步要做的是收口，不是扩散。
```
