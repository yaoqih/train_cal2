# 调车全流程策略执行版

> 当前四阶段代码的精确 ownership、线路释放证明、两批 assembly 和闭合多摘多挂语法，以
> [Stage1-Stage4 精确边界与闭合调车策略](./Stage1-Stage4精确边界与闭合调车策略.md) 为准。
> 本文的 P1-P5 是业务阶段，不等同于代码 Stage1-Stage4。

## 目标

这份文档不是统计总结，而是把人工计划和拓扑约束落成可执行策略。核心目标有三个：

1. 第一阶段先做会降低后续复杂度的前场整理和一部分存车塑形。
2. 不提前占用关键通路，不把 Stage4 的问题换成更隐蔽的路径阻塞。
3. Stage2/Stage3 分别闭合修库出库与大库入库；Stage4 保留大库完成后的前场目标重建、强制台位重排和尾项闭合。

策略必须按“线路角色状态”动态决策，不能按固定线路顺序硬排。

## 一句话边界

第一阶段处理“浅、干净、能解锁通路、能形成后续小列”的动作；第四阶段处理“深、脏、强闭合、强台位、多源重建”的动作。

更具体地说：

- 目标线干净、路径清晰：直接送，不绕到存4北或编组线。
- 目标线脏但北端阻塞浅：先清北端前缀，再送目标。
- 目标线脏且阻塞深，或需要多源重建：交给 Stage4。
- 临停线/前置线只有在下游债务关闭后才是资源线。
- 存4北是释放口，不是第一阶段半成品缓存。
- 机库线是人工计划高频承接资源，但必须按机库/称重/机区通路约束使用。

## 基础对象

### 车辆债务族

每辆车先归入一个主债务族，后续候选动作按债务族判断是否应该在当前阶段移动。

| 债务族 | 识别条件 | 当前阶段意义 |
|---|---|---|
| `front_service` | 目标含 `抛丸线/油漆线/洗罐站/洗罐线北` | 第一阶段优先关闭 |
| `beam_repair` | 目标含 `调梁棚/调梁线北` | 第一阶段只做浅层/解锁，主体留第二批或 Stage4 |
| `pre_repair` | 目标含 `预修线/机走棚` | 第一阶段只做浅层/塑形，深层留 Stage4 |
| `depot_inbound` | 目标含 `修1-4库内/库外` | 第一阶段只做成段承接，不做最终入库 |
| `store4_release` | 目标为 `存4线` 或已完成待出段 | 由 Stage2 出库/释放合同闭合 |
| `unwheel` | 目标含 `卸轮线` | 可作为前场服务处理，但和存4释放强相关 |
| `storage_rebalance` | 目标为 `存1/存2/存3/存5南` | P2 塑形或尾项处理 |
| `gate_blocker` | 当前挡住门槛线但自身目标不在该线 | 必须按真实目标或短闭合清走 |
| `satisfied_protected` | 已经满足目标且无必要移动 | 原则上不动，除非清障后原线恢复 |

如果一辆车同时满足多个族，按下面优先级解释：

```text
store4_release > depot_inbound > front_service > beam_repair > pre_repair > unwheel > storage_rebalance
```

但候选动作可以临时按“阻塞角色”提升优先级：挡住 `洗油北/洗罐线北/调梁线北/存5线北/修库库外` 的车，先按 `gate_blocker` 处理。

### 线路状态

每条线路在每个时刻都有一个策略状态：

| 状态 | 含义 | 能否放车 |
|---|---|---|
| `target_clean` | 目标线为空，或既有车都满足且无强制台位冲突 | 可直送目标车 |
| `target_dirty_shallow` | 目标线脏，但北端前缀可低成本清理 | 先清前缀再送 |
| `target_dirty_deep` | 目标线脏且前缀深、多源或强台位 | Stage4 重建 |
| `gate_locked` | 线路是下游目标的门槛，债务未关闭 | 禁止当缓存 |
| `gate_unlocking` | 当前动作正在清这个门槛 | 只允许清障动作 |
| `resource_unlocked` | 下游债务关闭，线路可短时承接 | 可作为编组/承接 |
| `release_port` | 正式释放口，如 `存4线` | 只允许释放或闭合中转 |
| `source_queue` | 入段源队列，如 `存5线北/南` | 只取不存，除非原线恢复 |
| `protected` | 已满足目标或强制位置稳定 | 不动或动后恢复 |

策略判断必须先更新线路状态，再生成候选动作。

## 全流程执行策略

### P0 起机和定位

P0 不产生真实车辆位移，但会决定第一钩从哪条线开始。

执行规则：

1. 机车先到能打开前场通路的源线。
2. 若存在 `抛丸/油漆/洗罐/机走棚/调梁棚/预修` 北端可取批次，优先从这些线开始。
3. 不从 `存5线北` 盲目开局，除非前场没有可做动作，或存5北端正好挡住前场目标。
4. 不从修库内开局，除非本案没有前场/存车塑形债务，或人工计划本身是纯库内释放。

P0 的输出是：

```text
first_work_area = front_service | beam_pre_repair | storage_queue | depot_closed
```

### P1 前场功能线整理

P1 是第一阶段最应该吸收的部分。它不追求最终全满足，而是打开通路、关闭浅层前场目标、形成后续承接小列。

#### P1 入口条件

满足任一条件就进入 P1：

- 有外部车要去 `抛丸线/油漆线/洗罐站/洗罐线北`。
- `抛丸线/油漆线/洗罐站/洗罐线北` 上有去大库、去存4或去其他线的车。
- `机走棚/预修线/调梁棚` 北端有能直接形成小列的目标车或阻塞车。
- `机南/洗油北/机走棚/洗罐线北/调梁线北` 任一门槛被当前车辆阻断。

#### P1 退出条件

同时满足以下条件时退出 P1：

- 抛丸即时债务关闭，`机南` 不再被抛丸目标依赖。
- 洗罐站、洗罐线北、油漆线的即时债务关闭，`洗油北` 不再是硬门槛。
- 前场线上去大库车已形成可取小段，或已经送入安全承接线。
- 未解决的调棚/预修债务要么深层，要么需要 Stage4 目标重建。

#### P1 候选动作优先级

按以下顺序生成并选择候选：

1. `direct_clean_target`
   - 目标线干净，路径不经过未解锁 gate。
   - 直接把目标车送到 `抛丸/油漆/洗罐/机库/预修/调棚`。

2. `clear_shallow_prefix`
   - blocker 能在同一 session 真目标落位或原线恢复，目标随后可一次关闭。
   - 只要缺少 blocker 退出、需要占未释放 gate 或已知还要重开目标，就不属于浅阻塞。

3. `service_line_extract_depot`
   - 从 `抛丸/油漆/洗罐/洗北` 抽出去大库车。
   - 只送到已释放、有 Stage3 owner 和退出路径的 assembly line；不再按静态线路顺序选择。

4. `front_mixed_train_to_jiku`
   - 连续挂取前场多源小列，送 `机库线` 承接。
   - 要求机库线可在后续 P4 或 P5 取出，不阻塞称重/机库目标。

5. `defer_deep_target`
   - 前缀深、多源、强制台位、路径需占未解锁 gate。
   - 不在 P1 做，标记给 Stage4。

#### P1 具体线路策略

抛丸：

```text
if exists external target 抛丸线:
    if 抛丸线 dirty:
        先清抛丸北端可取车
    送外部抛丸车到抛丸线
    禁止把抛丸去大库车默认甩到机南
else:
    清出抛丸去大库车
    可释放机南
```

洗罐：

```text
if target 洗罐站 and 洗罐线北 occupied by unrelated cars:
    先清洗罐线北
if 洗罐站 clean:
    直接送洗罐车
elif 洗罐站 shallow dirty:
    清北端前缀后送
else:
    留给 Stage4 target rebuild
在洗罐关闭前，洗油北禁止缓存
```

油漆：

```text
if 油漆线 clean and path clear:
    直接送油漆
elif 油漆线 shallow dirty:
    清前缀
else:
    后段处理
油漆未关闭前，不占洗油北
```

调棚/预修/机棚：

```text
if 北端可取车会阻断机南/洗油北/机区通路:
    可以在 P1 取出
elif 目标线 clean and 可直送小批:
    可以直送
else:
    不做完整重建，交给 P2/P4/P5 或 Stage4
```

### P2 存车分流塑形

P2 是第一阶段可以吸收的一部分，但不能变成“清空存5”。它的目标是塑形，不是全局求解。

#### P2 入口条件

- P1 已经关闭主要前场门槛，或当前没有可做 P1 动作。
- `存5线北/存5线南/存1/存2/存3` 北端有连续目标小段。
- 抽取该小段能直接送到干净目标线，或能形成后续大库/调棚/预修小列。

#### P2 退出条件

- 存5北端不再有低成本可抽小段。
- 下一批需要深挖存5南，且无关前缀无法在同一 session 真目标落位或恢复。
- 后续动作必须占用 `存4南/调梁线北/洗油北/机北2` 等未解锁 gate。
- 已形成第一批大库承接列，等待 P3/P4 闭合。

#### P2 抽取规则

从每条源线北端扫描：

```text
prefix = []
for car in access_order(source_line):
    family = debt_family(car)
    if car is protected_satisfied:
        break
    if family can be served now:
        prefix.append(car)
        continue
    if car blocks a higher-priority family and can go true target:
        prefix.append(car)
        continue
    break
```

允许抽取的批次：

- 同一目标族连续段。
- 混合但后续同一承接方向，例如都能进 `机库线` 后再拆。
- 前缀中阻塞车可以随主列带走，但必须在本阶段或下一阶段有明确落点。

拒绝抽取的批次：

- 为了取深层目标，越过大量无关车。
- 把洗油抛尾项夹到大库主列中间。
- 把调棚/预修尾项夹到修库入库主列中间。
- 抽出后只能占未解锁 gate。

#### P2 落点规则

按以下顺序选择落点：

1. 真实目标线干净：直接送。
2. `机库线` 可承接且后续能闭合取出：送机库线。
3. 已解锁临停/承接线：`机南/洗油北/机走棚/机走北`。
4. 存车目标线：`存2/存3/存1`，但不能破坏预修/存4南通路。
5. 都不满足：不抽，留给 Stage4 或尾项。

### P3 修库出库到存4

P3 不建议放在 Stage1。它是强闭合段，由当前代码 Stage2 处理。

#### P3 入口条件

- 修库内存在目标为 `存4线` 的车，或出段集结批次已经形成。
- 存4北释放口可用，存4南通路可用。
- 库外门槛可清，修1-4库内可按北端顺序取车。

#### P3 动作策略

1. 按修库内可取顺序取出出段车。
2. 取车过程中遇到卸轮/油漆等尾项，只有在不破坏主列顺序时顺手带出。
3. 出段主列在 20 折算辆、线路长度、关门车和尾端可摘顺序内尽量合并，不使用固定辆数目标。
4. `Put 存4线` 必须满足关门车位置、存4容量和北端释放要求。
5. 若库外有阻塞，清库外后必须恢复或纳入闭合链。

#### P3 禁止提前到 Stage1 的原因

- 会污染存4释放口。
- 会把修库出库和大库入库耦合到一起。
- 需要关门车、库外、库内顺序和存4北端位置多重约束。

### P4 大库入库重放

P4 是从承接线取已编好的去大库车，按修库目标连续摘解入库。

#### P4 入口条件

- 大库目标车已经在 `机库线` 或其他承接线上形成可摘小段。
- 修1-4库内/库外可达。
- 每个库目标车在机后顺序支持连续摘解。
- 称重车已经称重，或本次入库过程能合法经过称重。

#### P4 动作策略

1. 优先从 `机库线` 取主列。
2. 按列尾可摘顺序决定修库投放顺序，不硬编码修1到修4。
3. 每次 `Put 修X库内` 尽量一次放满该库连续段。
4. 若库外阻塞，先清库外，并把清出的车纳入同一闭合宏。
5. 若主列顺序不支持连续摘解，回退到承接线重排，不硬送。

#### P4 与机库线

`机库线` 的使用边界：

- P1/P2 可以把去大库和混合小列送入 `机库线`。
- 但送入前必须确认：没有立即需要称重/机库目标的车被阻断；`机北2/调梁线北` 通路不被长期占用；后续 P4 能取出。
- 如果机库线已经脏且无明确出路，不再继续塞车。

### P5 尾项回收

P5 不应完全靠搜索。它应该按尾项目标族生成小宏。

尾项优先级：

1. 存5尾项：从 `存5线北/南` 取剩余可直接服务小段。
2. 机库尾项：从 `机库线` 取剩余洗油抛、调棚、预修或大库小段。
3. 卸轮尾项：卸轮完成后去油漆、存4或大库。
4. 调棚/预修尾项：清剩余可直送小段。
5. 油漆/洗罐/抛丸尾项：最终补齐功能线。

P5 的目标不是减少一两钩，而是清掉临停污染：

```text
no long-lived cars on 机南/洗油北/存4南/机北1/机北2
no unresolved gate blockers
no satisfied car damaged
remaining unsatisfied only if explicitly out-of-scope
```

## Stage1 与下游阶段的精确边界

### Stage1 应该扩展到

Stage1 可以做以下动作：

1. P1 前场功能线整理。
2. P2 中低风险存车塑形。
3. 第一批去大库车辆的承接编组。
4. 干净目标线的直接送达。
5. 浅层目标线的北端前缀清理。
6. 为解锁 `机南/洗油北/洗罐线北/机走棚` 做的必要清障。

Stage1 的结束状态应满足：

```text
front_service_immediate_debt == 0 or only_deep_dirty_targets_left
critical_gates_locked_by_stage1 == 0
store4_pollution_by_depot_inbound == 0
assembly_batches_are_extractable == true
protected_satisfied_damage == 0
```

### Stage1 不应该做

1. 深层调棚重建。
2. 深层预修重建。
3. 修库出库到存4主闭合。
4. 大库最终入库。
5. 强制台位跨线重排。
6. 为深挖存5南移动大量无关前缀车。
7. 把半成品长期放在 `存4线/存4南/洗油北/机南/调梁线北/机北2`。

### Stage4 应该负责

1. Stage3 完成后的 P5/前场尾项回收宏。
2. 目标线脏深的 target rebuild。
3. route session 清障和恢复。
4. spotting/强制位置重排。
5. 洗北、调北、存5等前场硬门槛的闭合处理。
6. 存车、洗油抛、调梁、预修和机区残债。

修库库内/库外目标和卸轮目标分别由 Stage2/Stage3 上游合同拥有，Stage4 不重新规划。

### Stage4 不应该背的压力

如果 Stage1 留下以下状态，说明 Stage1 边界错了：

- 外部去抛丸还没处理，却把 `机南` 占了。
- 洗罐/油漆还没处理，却把 `洗油北` 占了。
- 调棚还要进出，却把 `调梁线北/机北2` 当缓存。
- 存4北被去大库半成品污染。
- 存5北端可直接分流的小段没有动，导致后面必须深挖。
- 前场干净目标线本可直送，却先绕进编组线。

## 运行时决策树

每一轮候选动作按这个顺序判断。

### 1. 更新债务和线路状态

```text
for each car:
    debt_family = classify_by_target_and_current_line(car)
    satisfied = target_satisfied(car)

for each line:
    target_clean = line_empty_or_all_satisfied(line)
    prefix_depth = first_unsatisfied_depth(line)
    gate_state = locked/unlocking/unlocked
```

### 2. 先找直送

```text
if exists batch whose target_line is clean
   and batch is north-accessible
   and route does not occupy locked gate
   and put will not break forced positions:
       choose direct_delivery
```

直送优先于编组，因为它直接减少债务且不制造临停污染。

### 3. 再找清前缀

```text
if target_line dirty
   and blockers can reach true targets or be restored in this session
   and target will not need a known reopen
   and all gate leases have a release step:
       clear prefix
       then re-evaluate direct_delivery
```

结构边界：

```text
D0 = direct_ready
D1 = one closed local cleanup session
D2 = dependency-complete joint session
D3 = missing source/gate/staging exit; do not move half-finished blocks
```

### 4. 再找前场释放和承接

```text
if service line contains depot_inbound cars
   and service target is closed or will be closed by this action:
       extract depot_inbound segment
       put to unlocked assembly/resource
```

资源选择：

```text
1. 机库线, if clean/extractable and no immediate weighing conflict
2. 已解锁机南
3. 已解锁洗油北
4. 已解锁机走棚/机走北
5. 存2/存3/存1 only for true storage or explicitly reversible staging
```

### 5. 再做存车塑形

```text
if source in 存5北/存5南/存1/存2/存3
   and north prefix forms useful group
   and target/resource is clean/unlocked:
       move prefix group
else:
       do not deep dig
```

### 6. 都不满足则交给对应下游 owner

```text
if remaining work requires:
    dirty deep target rebuild
    forced position insertion
    route blocker session
    multi-source repack
then stop Stage1 and invoke Stage4

if remaining work is depot outbound/store4 release:
    handoff Stage2

if remaining work is depot inbound:
    handoff Stage3 with assembly contract
```

## 人工动作模板

下面这些模板来自成功 Response 的高频相邻动作，不是固定作业单，而是求解器应优先尝试的结构模式。

### 开局模板

人工开局 84 个成功案例全部从有效 `Get` 开始，常见开局不是存5，也不是修库，而是前场功能线。

| 模板 | 策略含义 | 使用条件 |
|---|---|---|
| `Get 洗罐站 -> Get 油漆线` | 先挂洗/油侧小批，再统一送承接或目标 | 洗油侧通路清晰，批次不会压 `洗油北` |
| `Get 调梁棚 -> Put 机库线` | 调棚北端有可摘车，先移到机库承接 | `调梁线北/机北2` 通路可用，机库可闭合取出 |
| `Get 油漆线 -> Get 调梁棚 -> Put 油漆线` | 油漆目标可直接消化，顺手调整调棚北端 | 油漆线干净或浅脏，调棚只取北端小段 |
| `Get 机走棚 -> Put 机库线` | 清机棚前置，释放机南/洗油北方向 | 机走棚挡通路或有可承接去大库小段 |
| `Get 抛丸线 -> Get 存2线 -> Put 抛丸线` | 外部去抛丸车优先闭合 | 抛丸线未关闭，不能先占 `机南` |

开局选择规则：

```text
if front_service has clean/shallow target:
    start from front_service source
elif beam/pre line has north-accessible blockers:
    start from 调梁棚/预修/机走棚 and put to 机库 or true target
elif storage prefix blocks front target:
    start from storage prefix
else:
    defer to closed macro stage
```

### 机库承接模板

人工计划中 `Put 机库线` 后最常见的下一步是继续取 `预修线/调梁棚/存5线北`，说明 `机库线` 的角色是“承接后继续塑形”，不是普通终点。

| 模板 | 策略含义 | 使用条件 |
|---|---|---|
| `Get 预修线 -> Put 机库线` | 预修北端去大库/混合小段承接 | 预修不做深层重建，只取可摘段 |
| `Get 调梁棚 -> Put 机库线` | 调棚北端去大库/混合小段承接 | 调北通路可用，调棚内部不重建 |
| `Put 机库线 -> Get 预修线` | 机库暂存后继续清预修 | 机库未满且后续可取出 |
| `Put 机库线 -> Get 调梁棚` | 机库暂存后继续清调棚 | 不阻塞 `机北2/调梁线北` |
| `Put 机库线 -> Get 存5线北` | 承接前场后进入存车塑形 | P1 门槛已关闭，P2 可开始 |

机库线允许承接的批次：

- 去大库连续段，后续 P4 能整列取出。
- 去预修/调棚/洗油抛的小段，但必须在 P5 有明确尾项宏。
- 混合小列，前提是尾端可摘顺序不会破坏后续目标。

机库线拒绝承接的批次：

- 承接后没有可取路径，导致机库线变成脏终点。
- 承接后立即阻断称重或机库真实目标。
- 承接后必须长期占用 `机北2/调梁线北` 才能取出。

### 存5分流模板

`存5线北` 是 P2 主源线，但人工模式是“北端目标段抽取”，不是清空。

| 模板 | 策略含义 | 使用条件 |
|---|---|---|
| `Get 存5线北 -> Put 调梁棚` | 存5北端调棚段直送 | 调棚干净或浅脏，调北通路可用 |
| `Get 存5线北 -> Put 预修线` | 存5北端预修段直送 | 预修干净或浅脏，存2/渡7不被阻 |
| `Get 存5线北 -> Put 存2线` | 存车目标归位或过渡塑形 | 不阻断预修通路 |
| `Get 存5线北 -> Put 存3线` | 存车目标归位或给存4南让路 | 不影响存4南释放 |
| `Get 存5线南 -> Put 预修线` | 深层南端成段抽取 | 存5北前缀已清，南端批次足够 |

存5分流停止条件：

```text
if next target requires crossing unrelated prefix
   and that prefix has no same-session target/restore exit:
    stop P2
if put target is locked gate:
    stop P2
if extracted train cannot be tail-detached at next target:
    stop P2
```

### 存4闭合模板

人工中 `Put 存4线` 后 76 次马上 `Get 机库线`，这说明 `存4线` 是强阶段边界，不是临时缓存。

| 模板 | 策略含义 | 使用条件 |
|---|---|---|
| `Get 修库内... -> Put 存4线 -> Get 机库线` | 出段释放后立即转入大库入库 | 机库已有去大库主列 |
| `Get 修库内... -> Put 存4线 -> Get 存5线北` | 存4释放后处理存5尾项 | 机库无主列，存5有直接尾项 |
| `Put 存4线 -> Put 油漆/预修` | 少量顺手尾项 | 不能污染存4，且尾项真实目标明确 |

Stage1 只允许出现 `Put 存4线` 的两种情况：

```text
1. move cars true target is 存4线
2. same closed chain will immediately Get them out
```

否则 `Put 存4线` 应由 P3/P4 控制。

### 尾项模板

P5 末端最常落点是 `机库线` 和 `油漆线`。这说明尾项不是“剩什么做什么”，而是按目标族收束。

| 模板 | 策略含义 | 使用条件 |
|---|---|---|
| `Get 卸轮线 -> Put 油漆线` | 卸轮完成后进入油漆 | 油漆线可接收 |
| `Get 机库线 -> Put 油漆线` | 机库尾项送油漆 | 油漆线干净或浅脏 |
| `Put 调梁棚 -> Put 机库线` | 调棚尾项后机库收束 | 机库作为最终可接受状态或后续可取 |
| `Put 预修线 -> Put 机库线` | 预修尾项后机库收束 | 预修已关闭，机库可承接 |

尾项宏按目标族生成，不能让全局搜索随机碰：

```text
tail_sweep_order = 存5尾项 -> 机库尾项 -> 卸轮尾项 -> 调棚/预修尾项 -> 洗油抛尾项
```

## 策略判定矩阵

这一节把“能不能做”转成可以编码的判定表。

### 目标线判定

| 条件 | 策略动作 | 阶段 |
|---|---|---|
| 目标线空 | 直接送 | Stage1 优先 |
| 目标线全是已满足车且无强制位冲突 | 直接送到尾端可接位置 | Stage1 优先 |
| blocker 可真目标落位/原线恢复，且单一闭合 session 不会再次开目标 | 清理并送达 | Stage1 D1 |
| blocker 无本 session 退出，或清后仍要再次开目标 | 不清，等待前提 | Stage4 D2/D3 |
| 目标线有强制台位插入 | 不做普通 Put | Stage4 spotting/repack |
| 目标线多源混合且既有车不满足 | target rebuild | Stage4 |

### 源线判定

| 源线状态 | 策略动作 | 禁止 |
|---|---|---|
| 北端连续同目标族 | 整段取出 | 拆散连续段 |
| 北端混合但同承接方向 | 挂成小列送机库/真实目标 | 夹入大库主列中间 |
| 第一辆是已满足保护车 | 不动该线 | 为深层目标强行越过 |
| 前缀阻塞高优先目标 | 前缀按真实目标清走 | 清到未解锁 gate |
| 深层目标无法在同一 session 完成 blocker 处置和退出 | 停止当前阶段 | 盲目深挖 |

### 承接线判定

| 承接线 | 允许条件 | 禁止条件 |
|---|---|---|
| `机库线` | 承接段后续可取，且不挡称重/机库真实目标 | 无出路继续塞车 |
| `机南` | 抛丸关闭，且洗油侧无即时债务 | 外部去抛丸未处理 |
| `洗油北` | 洗北/洗南/油漆关闭 | 洗油抛未处理 |
| `机走棚` | 机棚/洗油/机南侧债务关闭 | 作为早期大库缓存 |
| `调梁线北` | 调棚不再进出 | 早期缓存 |
| `存2/存3/存1` | 真实存车目标或短闭合塑形 | 破坏预修/存4南/机北1通路 |
| `存4线` | 真实存4目标或同链中转 | 去大库半成品长期停放 |
| `存4南` | 仅通过或极短临停 | 任何普通缓存 |

### 阶段交接判定

| 当前状态 | 下一阶段 |
|---|---|
| 前场即时债务关闭，存5北端有可塑形段 | P2 |
| 前场剩余都是依赖闭包已齐的深脏目标 | Stage4 target-window rebuild |
| assembly contract 已形成 | Stage3 大库入库 |
| 存4释放口准备好且有出段主列 | Stage2/P3 |
| 大库入库完成但有机库/存5/卸轮小尾项 | P5 |
| 任一 locked gate 被阶段结果占用 | 回滚或继续清障，不能交接 |

## 质量检查清单

每个阶段结束前必须回答这些问题：

1. 是否还有外部去抛丸车？如果有，`机南` 是否保持可用。
2. 是否还有洗北/洗南/油漆车？如果有，`洗油北` 是否保持可用。
3. 是否还有调棚/机库即时债务？如果有，`调梁线北/机北2` 是否保持可用。
4. 是否有去大库半成品停在 `存4线/存4南`。
5. `机库线` 上的车是否能被后续整列或分段取出。
6. 存5北端是否还有可直接分流的浅层小段。
7. 已满足车辆是否被移动后没有恢复。
8. 当前主列是否支持下一目标尾端连续摘解。

只有这些检查通过，才允许阶段交接。

## 候选动作硬拒绝规则

以下动作不进入评分，直接拒绝：

1. 把车放到未解锁 `洗油北`。
2. 把车放到未解锁 `机南`，且还有外部去抛丸或洗油侧即时债务。
3. 把车放到未解锁 `调梁线北/机北2`，且还有调棚/机库即时债务。
4. 把非出段车长期留在 `存4线`。
5. 把任何车放到 `存4南` 作为缓存。
6. 目标线有强制台位冲突，还尝试普通 Put。
7. 移动已满足车后不能在同一闭合链恢复。
8. 为取深层车跨过超过阈值的无关前缀车。
9. 把尾项车夹入大库主列中间，导致后续不可连续摘解。
10. 路径需要经过有车的硬门槛，且没有同步清障计划。

## 候选动作评分

硬拒绝之后，按可解释分数排序：

```text
score =
    - 1000 * direct_target_satisfied_count
    -  700 * gate_unlocked_count
    -  500 * future_contiguous_run_gain
    -  300 * service_line_closed_count
    -  200 * depot_batch_extractable_gain
    +  400 * locked_gate_occupancy_count
    +  250 * dirty_target_put_count
    +  200 * prefix_depth_cost
    +  150 * route_detour_delta
    +  150 * future_fragment_count
    +  120 * storage_pollution_count
    +  100 * protected_car_moved_count
```

解释：

- `direct_target_satisfied_count`: 真实目标满足数量。
- `gate_unlocked_count`: 动作后 `机南/洗油北/洗北/调北/机棚` 等门槛从 locked 变 unlocked。
- `future_contiguous_run_gain`: 同目标族在承接线形成连续小段。
- `service_line_closed_count`: 抛丸/油漆/洗罐即时债务被关闭。
- `depot_batch_extractable_gain`: 去大库车在机库线或承接线形成可取段。
- `locked_gate_occupancy_count`: 动作后占用未解锁门槛。
- `dirty_target_put_count`: 普通 Put 到脏目标线。
- `future_fragment_count`: 同目标族被拆成更多段。

## 具体场景

### 场景 1：抛丸线上有去大库车，同时外部还有去抛丸车

错误策略：

```text
Get 抛丸线去大库车 -> Put 机南
```

原因：`机南` 是外部去抛丸常用路径的一段，会让后续去抛丸绕行或被阻。

正确策略：

```text
1. 判断外部去抛丸车是否北端可取。
2. 如果可取，先送外部去抛丸车到抛丸线。
3. 抛丸线关闭后，再抽抛丸去大库车。
4. 抽出的去大库车优先去机库线；只有抛丸关闭后才允许机南承接。
```

### 场景 2：洗罐站目标车在外部，洗罐线北有无关车

错误策略：

```text
直接找路去洗罐站，或把无关车甩洗油北
```

正确策略：

```text
1. 洗罐线北是洗罐站前置门槛，先处理洗北。
2. 洗北无关车能直归真实目标则直归。
3. 不能直归时，只能进短闭合承接，不能压洗油北。
4. 洗北清后再送洗罐站。
```

### 场景 3：油漆线干净，存5北端有油漆车

正确策略：

```text
直接从存5北取油漆车送油漆线。
```

不要先把这批车送机库、存4或机南，因为目标干净时直送就是最小债务、最少污染。

### 场景 4：调梁棚深层脏，但北端有去油漆/大库车

正确策略：

```text
1. 只取北端可摘的去油漆/大库/机库小段。
2. 去油漆若目标干净则直送。
3. 去大库小段送机库线承接。
4. 不在 Stage1 重建调梁棚内部顺序。
```

### 场景 5：存5北端连续一段去预修/调棚，后面夹杂去大库和洗罐

正确策略：

```text
1. 取北端连续同族段。
2. 去预修/调棚目标干净则直送。
3. 目标脏浅则清前缀再送。
4. 一旦需要跨过无关车深挖，停止 P2。
```

### 场景 6：去大库车已经在机库线成段

正确策略：

```text
1. Stage1 到此可以结束，不急着入库。
2. Stage4/P4 从机库线整列取出。
3. 按修库目标的尾端可摘顺序连续 Put 修库内。
```

### 场景 7：存4北空，手上有去大库车

错误策略：

```text
Put 存4线，阶段结束。
```

正确策略：

```text
只有两种允许：
1. 这批车真实目标是存4线。
2. 存4线只是闭合中转，并且同阶段马上 Get 走转入承接/入库。
```

### 新增策略状态

```python
LineStrategyState(

```

新增车辆策略标签：

```python
CarDebtTag(
    no: str,
    family: str,
    true_targets: tuple[str, ...],
    current_line: str,
    is_satisfied: bool,
    is_gate_blocker: bool,
    can_direct_deliver: bool,

```

### Stage1 生成器顺序

建议候选生成顺序改成：

```text
1. direct clean target
2. shallow prefix clear
3. front service closure
4. service-line depot extract
5. storage prefix shaping
6. unlocked resource assembly
7. stop and handoff to Stage2/Stage3/Stage4 owner
```

不要先生成“所有去大库车 -> ASSEMBLY_DEPOT”，否则会继续把压力转移给 Stage4。

### Stage2/Stage3/Stage4 宏边界

当前代码应按 ownership 拆成：

```text
Stage2: depot outbound / unwheel / store4 release
Stage3: depot inbound from assembly contracts
Stage4: target-window rebuild / route session / tail sweep for remaining front targets
```

尤其要把 P5 尾项宏显式化，而不是只靠全局候选搜索。

## 验收指标

策略实现后不只看总钩数，要看这些指标：

| 指标 | 期望变化 |
|---|---|
| Stage4 `put_route_blocked` | 下降 |
| Stage4 `active_blocked_by_fixed` | 下降 |
| Stage4 最终 unsatisfied | 下降 |
| Stage1 结束时未解锁 gate 占用 | 归零或显著下降 |
| `存4线` 非出段污染 | 归零 |
| `机南/洗油北/调梁线北/机北2` 长期占用 | 归零 |
| 机库线大库成段质量 | 上升 |
| 前场干净目标直送率 | 上升 |
| replay 物理违规 | 不能上升 |

## 最终策略摘要

实际调车策略应当是：

```text
先前场，后存车；
先解锁，后编组；
先直送干净目标，后清浅阻塞；
先保护通路，再借用临停；
先形成有 owner 和退出路径的可取小列，再交给 Stage3 大库闭合；
存4只做释放口，或在同一原子 session 内短时预编后恢复；
深层调棚、预修和前场强台位交给 Stage4；库内交给 Stage2/Stage3。
```

这比“第一阶段多搬一点车”更严格。第一阶段搬错车会让第四阶段更难；第一阶段真正要做的是让第四阶段面对更少的门槛、更好的车列顺序和更干净的目标线。
