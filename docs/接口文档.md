# 调车计划生成 WebAPI 接口文档


---

## 1. 生成调车计划

* **功能描述**：根据当前站内车辆的初始位置、目标线路及各物理股道的配置，返回调车机车的详细作业指导步骤（勾计划）及计算后的实际对位结果。
* **请求路径**：`/api/plan/generate`
* **请求方式**：`POST`
* **Content-Type**：`application/json`

### 1.1 请求参数 (Request Body)

请求体需传递一个 JSON 对象，包含2个核心列表：`StartStatus`（起点状态）和 `TerminalLines`（库内股道配置）。

#### 根对象 (`PlanCreatePostData`)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `StartStatus` | `List<StartCarDto>` | 是 | 车辆的初始状态列表。 |
| `TerminalLines` | `List<TerminalLineDto>` | 是 | 终点规划器可用的目标股道配置信息。 |
| `locoNode` | `LocoNodeDto` | 是 | 调车机当前位置，包含股道全称和南北端。 |

#### `LocoNodeDto` (调车机位置)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `Line` | string | 是 | 调车机所在股道全称，例如：`"机库线"`。 |
| `End` | string | 是 | 调车机所在端：`"North"` 或 `"South"`。 |

#### `StartCarDto` (初始状态)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `Line` | string | 是 | 起点股道全称，例如：`"修1库外"`。 |
| `Position` | int | 是 | 起点连续台位号，非负整数。 |
| `RepairProcess` | string | 是 | 修程["厂", "段", "临", "其他"]  |
| `Type` | string | 是 | 车型信息。 |
| `No` | string | 是 | 车号。 |
| `Length` | decimal | 是 | 车辆长度，单位m |
| `IsHeavy` | bool | 否 | 是否为重车，默认 `false`。 |
| `IsWeigh` | bool | 否 | 是否需要称重，默认 `false`。 |
| `IsClosedDoor` | bool | 否 | 是否是关门车，默认 `false`。 |
| `TargetLines` | `List<string>` | 是 | 目标股道全称。如有多个候选线，可传多个股道，例如：`["修1库内", "修2库内"]`。 |
| `ForceTargetPosition` | `List<int>` | 否 | 强制对位台位号。修1-修4库内按绝对库位判断；调梁棚、洗罐站、油漆线、抛丸线按北向编号定义对位窗口的北侧下限和容量，实际达标按南端车辆占用情况允许窗口向南浮动，但容量不扩大。例如调梁棚总台位 11，`6-9` 表示 4 个对位台位；南端无非对位车时单车落点可在 `6-11` 内浮动，但同组对位车仍最多 4 辆，且必须能共同落入一个连续 4 台位窗口，如 `8-11`、`7-10` 或 `6-9`；南端有 3 辆非对位车时可用落点为 `6-8`。非对位车可以占在对位窗口内，剩余空位仍可放对位车，不会因窗口内混入非对位车而整段失效。台位号只用于相对位置窗口，容量仍按线路有效长度判断。除修1-修4库内允许多个目标股道外，其他线路启用强制对位时只能指定一条 `TargetLines`。 |

#### `TerminalLineDto` (库内股道配置)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `Line` | string | 是 | 修1-修4库内股道全称，例如：`"修1库内"`。 |
| `IsInspectionMode` | bool | 否 | 该库内线是否按迎检模式开放。请求中存在迎检目标时，库内台位使用迎检范围 7 位（否则使用常规 5 位）。 |

---

### 1.2 响应参数 (Response Body)

返回标准响应结构 `PlanGenerateResponseDto`，核心数据在 `Data` 字段中。

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `Success` | bool | 业务处理是否成功。 |
| `Message` | string | 提示信息或异常信息。 |
| `StatusCode` | int | HTTP 状态码。 |
| `Data` | `PlanGenerateResultDto` | 处理成功时返回的数据载荷。 |

#### `PlanGenerateResultDto`

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `Operations` | `List<PlanOperationDto>` | 调车机作业步骤。 |
| `GeneratedEndStatus` | `List<GeneratedEndCarDto>` | 计算后的最终对位结果。 |

#### `PlanOperationDto` (动作信息，提取关键字段)

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `Index` | int | 动作序号。 |
| `Line` | string | 展示用完整股道名（如 `"修1库内"`, `"存5线北"`）。 |
| `Action` | string | 操作动作：`Get` (取车), `Put` (放车), `Weigh` (称重)。 |
| `MoveCars` | `List<string>` | 参与本次移动的车号列表。 |
| `TrainCars` | `List<string>` | 操作后机车上承载的车号列表。 |
| `PassbyPath` | `List<string>` | 从上一勾机车位置到当前勾经过的线路和道岔列表，包含起点和终点；股道输出全称，道岔/节点输出原始编号，如：`["存1线", "L5", "L6", "Z1", "L8", "L9", "洗罐线北"]`。 |

`Action=Weigh` 时，`TrainCars` 是本勾挂着经过机库的整组车；`MoveCars` 只填本次实际标记为称重完成的车。若一勾挂了多辆称重车，只有编组最后一辆需要称重的车在本次 `Weigh` 中完成称重。

#### `GeneratedEndCarDto`

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `No` | string | 车号。 |
| `Line` | string | 最终所在股道全称。 |
| `Position` | int | 最终台位号。 |

### 1.3 最小输入输出示例

下面示例只保留最小必要字段，用来说明请求和响应的基本结构；实际返回中 `PassbyPath` 会随真实求解路径变化。

#### 最小输入示例

```json
{
  "StartStatus": [
    {
      "Line": "存5线北",
      "Position": 1,
      "RepairProcess": "段",
      "Type": "C70",
      "No": "1000001",
      "Length": 14.3,
      "TargetLines": ["机库线"]
    }
  ],
  "TerminalLines": [
    {
      "Line": "修1库内",
      "IsInspectionMode": false
    }
  ],
  "locoNode": {
    "Line": "机库线",
    "End": "North"
  }
}
```

#### 最小输出示例

```json
{
  "Success": true,
  "Message": "",
  "StatusCode": 200,
  "Data": {
    "Operations": [
      {
        "Index": 1,
        "Line": "存5线北",
        "Action": "Get",
        "MoveCars": ["1000001"],
        "TrainCars": ["1000001"],
        "PassbyPath": ["机库线", "L7", "L6", "L5", "L3", "L1", "L2", "存5线北"]
      },
      {
        "Index": 2,
        "Line": "机库线",
        "Action": "Put",
        "MoveCars": ["1000001"],
        "TrainCars": [],
        "PassbyPath": ["存5线北", "L2", "L1", "L3", "L5", "L6", "L7", "机库线"]
      }
    ],
    "GeneratedEndStatus": [
      {
        "No": "1000001",
        "Line": "机库线",
        "Position": 1
      }
    ]
  }
}
```
