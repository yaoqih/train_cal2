# Stage3

Stage3 从 Stage2 终态接管去大库车辆，完成修1-4库内/库外的线路与台位分配。

## 模型

1. `placement.py` 把每辆车的库内台位、库外线路和混合目标放进同一个有限域。
   约束同时覆盖稀疏强制台位、固定车访问前沿、厂段修顺序、库外长度和恢复车占用。
2. 根节点先检查库内 Hall 缺集和库外加权子集容量。不可行结果携带车辆集、
   可达位置集、需求、容量和缺口，可独立复算。
3. `transactions.py` 在稳定闭包上搜索。每个事务必须严格增加已稳定对位车辆。库内
   与强制槽车辆保持线路和绝对台位；普通紧凑外库车辆保持线路及已提交相对顺序。
4. 容量交换、任意深度库门链、台位对齐、单步直接落位和普通合法动作属于同一事务候选集合；
   普通邻接保留所有通过物理检查的 Get 前缀和 Put 后缀，不用目标启发式提前删边。
   外层队列先比较严格单调的未提交车辆数，再用库内源线阻塞、待完成内库台位、
   临时线错位和牵出列长度打破同进度平局，避免深闭包状态被浅层低障碍状态饿死。
   通用邻接预算用尽后，队列最多再展开最优先的 2 个有效非终态，并允许识别它们
   产生的完整后继；此时只消费无需局部搜索的直接落位、容量交换和库门链，台位对齐
   与通用邻接同时停止。预算后只允许把结果升级为 complete，额外 partial 进度不参与
   构型排名。所有事务仍通过同一闭包/投影不变量，不会切换求解器。
5. 台位对齐以“牵出列严格缩短”为单调量，枚举尾部切分和全部合法临时线；每个
   待对齐内库最多扩展 256 个局部状态，并在内部检查全局 deadline。该局部上限与
   通用事务扩展预算分别计数；截断会显式标记搜索空间未完成。
6. 跨模板先按固定 template/layout 的可采纳业务勾下界排序，再使用 placement 代理分；
   代理分在距离偏好之前惩罚“暂存车占用仍有活动内库需求的配对外门”，补齐人工的
   门位保留意识。代理分不能把另一模板完全挤出中层预算，也不能解释为最优性距离。
7. placement 候选先按 25 次通用事务扩展预算依次试探；首个完整解出现后只完成
   同下界/代理分组。浅层完整解未达到 2 个时，筛选层 6 个席位由 4 个下界/构型/
   模板覆盖席位和 2 个浅层闭包进度席位组成。若未提前得到足够完整解或严格改进，
   六个构型各自至多公平加深至 100，防止单个高进度慢构型耗尽全局时间；仍无
   完整解时，finalist 先保留静态排名最好的 2 个不同 assignment 签名，再加入其余
   候选中预算内搜索势第一名。前两个静态代表已有 complete 时不再运行第三席；否则
   三席依次加深至 5000 和 10000，避免单一代理独占深层预算。
8. 同一构型只保留支配结果：完整优先；partial 先比较同一扩展预算内曾到达状态的
   最优 `(剩余车辆, 错误库内源线, 待完成内库台位, 临时线错位, 牵出列长度)`，再
   比较尝试业务勾数和原子操作数。完整解按实际业务勾数择优；代理 placement 分只
   排序，不能提前证明优劣。
9. 车辆行数据和线路映射缓存只在一次 layout 搜索内存活；跨 layout 清理 State 强引用，
   路由、assignment 和 pickup 缓存仍按完整物理键或 `(template, layout)` 安全复用。
10. 物理层逐步验证路径、操作端、牵引、容量、库门和最终 `Positions`。

完整结果必须通过 Stage3 独立 replay、Stage1-3 合并 replay、台位规则和内部状态
投影。主路径只有这一套模型；预算耗尽、已证明不可行和程序异常具有不同语义，
不存在失败后切换的备用求解器。

## 运行

```bash
python3 scripts/stage3_simple/solve.py data/truth2 \
  --stage2-out artifacts/fullflow_current/truth2/stage2 \
  --out artifacts/fullflow_current/truth2/stage3
```

`input` 传 JSON 文件时运行单案，传目录时批量运行全部 `validation_*.json`。
输入不存在或目录中没有案例时直接报错。Stage2 产物缺失或未完成记为
`unavailable`，Stage3 在显式预算内未完成记为 `partial`。程序异常保留 traceback
并立即终止批次，不会降级成普通不可解案例。批次中存在非 complete 案例时退出码为 1。
`partial` 会报告内部尝试勾数和稳定闭包车辆数，但响应的 `Operations` 与
`GeneratedEndStatus` 始终为空，不能把未完成事务交给下游执行。
